# MVP 1단계 작업 Command

당신은 1인칭 시점의 2D 멀티플레이 심리 추리 액션 게임을 개발하는 역할입니다.

코드를 수정하기 전에 먼저 아래 문서를 읽으세요.

- docs/game-concept.md
- docs/mvp-scope.md
- docs/technical-direction.md

위 문서들을 프로젝트의 기준 문서로 사용하세요.

## 프로젝트 요약

이 프로젝트는 React, Next.js, TypeScript, Phaser 기반의 1인칭 시점 2D 멀티플레이 웹 게임입니다.

추후 WebView로 감싸 iOS/Android 모바일 앱 게임으로 확장할 예정입니다.

핵심 플레이는 많은 NPC 사이에 실제 플레이어가 숨어 있고, 술래가 NPC 무리 속에서 실제 참여자를 찾아내는 것입니다.

참여자는 NPC처럼 행동하며 살아남아야 하고, 술래는 NPC를 잘못 공격하지 않으면서 실제 참여자를 찾아 공격해야 합니다.

## 공통 규칙

작업 중 아래 규칙을 반드시 지키세요.

- 전체 게임을 한 번에 구현하지 않습니다.
- 현재 요청한 MVP 단계의 범위만 구현합니다.
- 아직 실시간 서버를 구현하지 않습니다.
- 아직 데이터베이스를 구현하지 않습니다.
- 로그인, 결제, 랭킹, 상점, 스킨, 친구 초대, 매치메이킹, 티켓 구매 시스템을 구현하지 않습니다.
- 대규모 N:N 팀전은 아직 구현하지 않습니다.
- 구현은 작고 확장하기 쉽게 유지합니다.
- Phaser 게임 로직과 Next.js 페이지 코드를 분리합니다.
- Phaser 코드는 반드시 클라이언트에서만 실행되게 합니다.
- Phaser로 인한 SSR 문제를 피합니다.
- 게임 엔티티와 역할 상태는 TypeScript 타입으로 분리합니다.
- mock 데이터는 별도 파일로 분리합니다.
- 게임 상태는 추후 실시간 서버 상태로 교체할 수 있도록 구조화합니다.
- 구현 후 변경 파일, 실행 방법, 확인 방법을 정리합니다.

## MVP 1단계 목표

이번 작업에서는 MVP 1단계만 구현합니다.

MVP 1단계 목표는 다음과 같습니다.

- Next.js + TypeScript + Phaser 프로젝트 구조를 만들거나 정리합니다.
- `/game` 페이지를 추가합니다.
- `/game` 페이지에서 Phaser 게임 캔버스를 렌더링합니다.
- 로컬 mock 데이터로 아래 엔티티를 표시합니다.
  - player 1명
  - seeker 1명
  - NPC 여러 명
- 각 역할이 시각적으로 구분되게 표시합니다.
- 간단한 2D 맵 또는 배경을 표시합니다.
- 1인칭 시점 카메라를 적용합니다 (자신의 캐릭터는 화면에 표시하지 않습니다).
- 기본 렌더링에 꼭 필요하지 않다면 이동은 구현하지 않습니다.
- 공격은 아직 구현하지 않습니다.
- 미션은 아직 구현하지 않습니다.
- 실시간 멀티플레이는 아직 구현하지 않습니다.
- DB 저장은 아직 구현하지 않습니다.

## 작업 방식

### Step A: 문서 읽고 요약하기

먼저 아래 문서를 읽으세요.

- docs/game-concept.md
- docs/mvp-scope.md
- docs/technical-direction.md

그다음 아래 항목을 요약하세요.

- 게임의 핵심 콘셉트
- MVP 1단계에 포함되는 기능
- MVP 1단계에서 제외되는 기능
- Next.js와 Phaser의 역할 분리
- 구현 시 주의해야 할 점

이 단계에서는 파일을 수정하지 마세요.

### Step B: 작업 계획 제안하기

문서 요약 후 MVP 1단계 구현 계획을 제안하세요.

아래 항목을 포함하세요.

- 작업 범위
- 제외 범위
- 예상 파일 구조
- 구현 순서
- 리스크 또는 결정이 필요한 부분

이 단계에서도 파일을 수정하지 마세요.

### Step C: 승인 기다리기

파일을 수정하기 전에 반드시 명시적인 승인을 기다리세요.

아래와 같은 승인 표현이 있을 때만 구현을 시작하세요.

- "진행해"
- "구현해"
- "좋아 진행"
- "Proceed"
- "Implement it"
- "Go ahead"

### Step D: MVP 1단계 구현하기

승인을 받은 뒤 MVP 1단계만 구현하세요.

구현 방향은 다음과 같습니다.

- `/game` 라우트는 클라이언트 전용 Phaser 컨테이너를 렌더링합니다.
- Phaser 게임 인스턴스는 컴포넌트가 마운트될 때 한 번 생성합니다.
- 컴포넌트가 언마운트될 때 Phaser 게임 인스턴스를 정리합니다.
- mock 게임 엔티티는 별도 mock 데이터 파일에서 불러옵니다.
- 엔티티 타입은 별도 타입 파일로 정의합니다.
- Phaser Scene은 간단한 맵과 엔티티를 렌더링합니다.
- player, seeker, NPC는 시각적으로 구분되어야 합니다.
- 이후 단계에서 이동, 미션, 공격, 실시간 상태를 추가할 수 있는 구조로 만듭니다.

### Step E: 검증하기

구현 후 프로젝트를 검증하세요.

아래 항목을 확인하세요.

- `/game` 페이지가 정상적으로 렌더링되는지
- Phaser 캔버스가 표시되는지
- hot reload 또는 remount 시 Phaser 캔버스가 중복 생성되지 않는지
- mock player, seeker, NPC가 화면에 보이는지
- TypeScript 타입 에러가 없는지
- 가능한 경우 lint/build 에러가 없는지 확인하고 수정했는지

### Step F: 결과 보고하기

검증 후 아래 내용을 정리해서 보고하세요.

- 변경한 파일 목록
- 구현한 내용
- 프로젝트 실행 방법
- `/game` 페이지 확인 방법
- 의도적으로 아직 구현하지 않은 기능
- 추천하는 MVP 2단계 작업
