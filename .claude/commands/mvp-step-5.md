# MVP 5단계 작업 Command

당신은 1인칭 시점의 3D 멀티플레이 심리 추리 액션 게임을 개발하는 역할입니다.

코드를 수정하기 전에 먼저 아래 문서를 읽으세요.

- docs/game-concept.md
- docs/mvp-scope.md
- docs/technical-direction.md
- docs/mvp-roadmap.md

위 문서들을 프로젝트의 기준 문서로 사용하세요.

## 프로젝트 배경

MVP 4단계에서 seeker 공격, hider 피격 시 탈락, NPC 오공격 시 seeker 체력 감소, hider 근접 경고/기여도 HUD를 추가했습니다. 이번 MVP 5단계는 그 위에서 미션과 미션 실패 시 강제 변신을 추가합니다.

게임 자체의 콘셉트/역할/규칙은 위에서 읽은 `docs/game-concept.md`(특히 "Mission System" 절), `docs/mvp-scope.md`를 기준으로 삼으세요 (여기서 다시 요약하지 않습니다).

기존 역할 전환 디버그 기능(`src/game3d/debug/*`)과 조작 역할별 HUD 분리 패턴(`src/game3d/combat/CombatHud.tsx`)을 그대로 재사용하세요. 미션/변신 관련 정보도 조작 중인 역할(`controlledRole`)에 따라 다르게 보여야 합니다 (미션 타이머·강제 변신 상태는 hider 조작 중일 때만 의미가 있습니다).

## 공통 규칙

작업 중 아래 문서의 규칙을 따르세요.

- `CLAUDE.md` — 클라이언트 전용 렌더링, 게임 엔티티/mock 데이터 분리 구조
- `docs/mvp-scope.md`의 `Excluded` 목록
- `docs/technical-direction.md`의 "Future Considerations" (특히 스토리텔링/콘텐츠 확장 항목 — 미션 종류/텍스트는 하드코딩하지 말고 데이터로 분리)

추가로 아래 두 가지는 이번 커맨드에 고유한 규칙이니 항상 지키세요.

- 전체 게임을 한 번에 구현하지 않고, 현재 요청받은 MVP 단계의 범위만 구현합니다.
- 구현 후 변경 파일, 실행 방법, 확인 방법을 정리합니다.

## MVP 5단계 목표

이번 작업에서는 MVP 5단계만 구현합니다. (`docs/mvp-roadmap.md` 기준 "미션과 강제 변신이 동작한다" 단계입니다.)

MVP 5단계 목표는 다음과 같습니다.

- 일정 주기로 hider에게 미션이 주어집니다.
- hider가 제한 시간 안에 미션을 수행하면 성공 처리됩니다.
- hider가 미션을 실패하면 5초 동안 랜덤한 다른 생명체(외형)로 강제 변신합니다.
- 변신 상태가 화면에서 명확히 구분됩니다 (변신 중에도 이동은 가능합니다).
- 변신 시간이 끝나면 원래 모습으로 돌아옵니다.
- 미션 타이머/현재 미션/변신 상태는 hider를 조작 중일 때만 HUD에 표시됩니다.
- 미션 종류는 `docs/game-concept.md`의 예시 중 **현재 입력 체계(이동)만으로 판정 가능한 것부터** 최소한으로 시작합니다 (예: 특정 구역으로 이동하기, 제자리에 머무르기, 점프). 그 외의 새 입력이 필요한 미션은 이후 단계로 미룹니다.
  - 점프는 스페이스로 가능합니다.
- NPC가 미션과 같은 랜덤 행동을 수행해 hider의 위장을 돕는 것은 **이번 단계에서는 구현하지 않습니다** (제외 목록 참고).
- 복잡한 미션 종류, 고급 연출, 서버 기반 미션 동기화, 재화 보상은 아직 구현하지 않습니다.
- 최종 승패 플로우, 실시간 서버 판정은 아직 구현하지 않습니다.

## 작업 방식

### Step A: 문서 읽고 요약하기

먼저 아래 문서를 읽으세요.

- docs/game-concept.md
- docs/mvp-scope.md
- docs/technical-direction.md
- docs/mvp-roadmap.md

그다음 아래 항목을 요약하세요.

- 게임의 핵심 콘셉트
- MVP 4단계에서 이미 구현된 내용 (현재 코드 기준)
- MVP 5단계에 포함되는 기능
- MVP 5단계에서 제외되는 기능
- Next.js와 Three.js의 역할 분리
- 구현 시 주의해야 할 점

이 단계에서는 파일을 수정하지 마세요.

### Step B: 작업 계획 제안하기

문서 요약 후 MVP 5단계 구현 계획을 제안하세요.

아래 항목을 포함하세요.

- 작업 범위
- 제외 범위
- 예상 파일 구조(신규/수정 파일 — 미션 데이터/타입, 미션 타이머, 변신 상태/외형)
- 구현 순서
- 리스크 또는 결정이 필요한 부분 (예: 이번 단계에 포함할 미션 종류, 미션 판정 방식, 강제 변신 시 외형 표현 방식, 미션 발생 주기/제한 시간 값)

이 단계에서도 파일을 수정하지 마세요.

### Step C: 승인 기다리기

파일을 수정하기 전에 반드시 명시적인 승인을 기다리세요.

아래와 같은 승인 표현이 있을 때만 구현을 시작하세요.

- "진행해"
- "구현해"
- "좋아 진행"
- "Proceed"
- "Implement it"
- "Go ahead"

### Step D: MVP 5단계 구현하기

승인을 받은 뒤 MVP 5단계만 구현하세요.

구현 방향은 다음과 같습니다.

- 기존 MVP 4단계 구조(`src/game3d/*`)를 최대한 재사용하고 확장합니다.
- 미션 종류/텍스트는 코드에 하드코딩하지 않고 별도 데이터(타입 + 목록)로 분리해, 이후 미션이 추가되어도 로직을 건드리지 않고 확장할 수 있게 합니다.
- 미션 주기/제한 시간/강제 변신 지속 시간은 상수로 분리합니다.
- 미션 성공/실패, 강제 변신 상태는 이후 실시간 서버 상태로 교체할 수 있도록 구조화합니다.
- 강제 변신 중에도 기존 이동 로직은 그대로 동작해야 합니다.
- NPC 랜덤 미션 행동, 복잡한 미션 종류, 승패 플로우, 실시간 서버 판정은 이번 단계에서 구현하지 않습니다.
- 이후 단계(NPC 미션 행동, 승패 플로우)를 추가할 수 있는 구조로 만듭니다.

### Step E: 검증하기

구현 후 프로젝트를 검증하세요.

아래 항목을 확인하세요.

- `/game` 페이지가 정상적으로 렌더링되는지
- hider 조작 중일 때 미션이 주기적으로 발생하고 HUD에 표시되는지
- 제한 시간 안에 미션 조건을 만족하면 성공 처리되는지
- 제한 시간 안에 미션을 수행하지 못하면 5초간 강제 변신이 발동하는지
- 변신 중에도 이동이 가능한지
- 변신 시간이 끝나면 원래 모습으로 돌아오는지
- seeker 조작 중일 때는 미션/변신 관련 HUD가 표시되지 않는지
- 디버그 역할 전환, 기존 이동/공격 기능이 여전히 정상 동작하는지
- hot reload 또는 컴포넌트 remount 시 3D 캔버스/렌더러가 중복 생성되지 않는지
- TypeScript 타입 에러가 없는지
- 가능한 경우 lint/build 에러가 없는지 확인하고 수정했는지

### Step F: 결과 보고하기

검증 후 아래 내용을 정리해서 보고하세요.

- 변경한 파일 목록
- 구현한 내용
- 프로젝트 실행 방법
- `/game` 페이지 확인 방법 (미션 발생/성공/실패/강제 변신 확인 방법 포함)
- 의도적으로 아직 구현하지 않은 기능
- 추천하는 MVP 6단계 작업
