# MVP 2단계 작업 Command

당신은 1인칭 시점의 3D 멀티플레이 심리 추리 액션 게임을 개발하는 역할입니다.

코드를 수정하기 전에 먼저 아래 문서를 읽으세요.

- docs/game-concept.md
- docs/mvp-scope.md
- docs/technical-direction.md
- docs/mvp-roadmap.md

위 문서들을 프로젝트의 기준 문서로 사용하세요.

## 프로젝트 배경

MVP 1단계에서 Phaser를 제거하고 Three.js(`@react-three/fiber`) 기반 3D 렌더링 파이프라인을 구축했습니다 (빈 씬 + 바닥 + 조명). 이번 MVP 2단계는 그 위에서 mock 데이터 기반 player/seeker/NPC를 3D 모형으로 표시하고, 1인칭 시점 카메라(자신의 캐릭터는 미표시)를 적용합니다.

게임 자체의 콘셉트/역할/규칙은 위에서 읽은 `docs/game-concept.md`, `docs/mvp-scope.md`를 기준으로 삼으세요 (여기서 다시 요약하지 않습니다).

## 공통 규칙

작업 중 아래 문서의 규칙을 따르세요.

- `CLAUDE.md` — 클라이언트 전용 렌더링, 게임 엔티티/mock 데이터 분리 구조
- `docs/mvp-scope.md`의 `Excluded` 목록
- `docs/technical-direction.md`의 "Future Considerations"

추가로 아래 두 가지는 이번 커맨드에 고유한 규칙이니 항상 지키세요.

- 전체 게임을 한 번에 구현하지 않고, 현재 요청받은 MVP 단계의 범위만 구현합니다.
- 구현 후 변경 파일, 실행 방법, 확인 방법을 정리합니다.

## MVP 2단계 목표

이번 작업에서는 MVP 2단계만 구현합니다. (`docs/mvp-roadmap.md` 기준 "화면에 게임이 뜬다" 단계입니다.)

MVP 2단계 목표는 다음과 같습니다.

- mock 데이터 기반으로 player 1명, seeker 1명, NPC 여러 명이 3D 모형(초기에는 primitive)으로 표시됩니다.
- 각 역할은 형태 또는 색으로 시각적으로 구분됩니다.
- 1인칭 시점 카메라가 적용됩니다 (자신의 캐릭터는 화면에 표시되지 않습니다).
- 카메라와 캐릭터 위치는 아직 고정 값입니다 (이동은 다음 단계에서 구현합니다).
- 시야 회전은 아직 구현하지 않습니다.
- 공격, 미션, 실시간 서버 동기화, 고급 NPC AI, DB 저장은 아직 구현하지 않습니다.

## 작업 방식

### Step A: 문서 읽고 요약하기

먼저 아래 문서를 읽으세요.

- docs/game-concept.md
- docs/mvp-scope.md
- docs/technical-direction.md
- docs/mvp-roadmap.md

그다음 아래 항목을 요약하세요.

- 게임의 핵심 콘셉트
- MVP 1단계에서 이미 구축된 3D 렌더링 파이프라인 (현재 코드 기준)
- MVP 2단계에 포함되는 기능
- MVP 2단계에서 제외되는 기능
- Next.js와 Three.js의 역할 분리
- 구현 시 주의해야 할 점

이 단계에서는 파일을 수정하지 마세요.

### Step B: 작업 계획 제안하기

문서 요약 후 MVP 2단계 구현 계획을 제안하세요.

아래 항목을 포함하세요.

- 작업 범위
- 제외 범위
- 예상 파일 구조(신규/수정 파일 — 엔티티 타입, mock 데이터, 렌더링 컴포넌트)
- 구현 순서
- 리스크 또는 결정이 필요한 부분 (예: primitive 형태/색상으로 역할 구분하는 방식, 1인칭 카메라에서 자기 캐릭터를 숨기는 방식)

이 단계에서도 파일을 수정하지 마세요.

### Step C: 승인 기다리기

파일을 수정하기 전에 반드시 명시적인 승인을 기다리세요.

아래와 같은 승인 표현이 있을 때만 구현을 시작하세요.

- "진행해"
- "구현해"
- "좋아 진행"
- "Proceed"
- "Implement it"
- "Go ahead"

### Step D: MVP 2단계 구현하기

승인을 받은 뒤 MVP 2단계만 구현하세요.

구현 방향은 다음과 같습니다.

- 기존 MVP 1단계 구조(`src/game3d/GameScene.tsx`)를 최대한 재사용하고 확장합니다.
- 게임 엔티티(player/seeker/npc) 타입과 mock 데이터를 별도 파일로 분리합니다.
- 각 엔티티는 역할에 따라 형태 또는 색으로 구분되는 primitive 3D 모형으로 렌더링합니다.
- 1인칭 시점 카메라를 적용하되, player 자신의 모델은 렌더링하지 않습니다.
- 카메라/캐릭터 위치는 아직 고정 값으로 둡니다 (이동은 다음 단계에서 구현합니다).
- 공격, 미션, 실시간 상태, 고급 AI는 이번 단계에서 구현하지 않습니다.
- 이후 단계(이동, 공격, 미션)를 추가할 수 있는 구조로 만듭니다.

### Step E: 검증하기

구현 후 프로젝트를 검증하세요.

아래 항목을 확인하세요.

- `/game` 페이지가 정상적으로 렌더링되는지
- player, seeker, NPC가 화면에 표시되는지
- 각 역할이 형태 또는 색으로 구분되는지
- player 자신의 모델이 화면에 보이지 않는지
- hot reload 또는 컴포넌트 remount 시 3D 캔버스/렌더러가 중복 생성되지 않는지
- TypeScript 타입 에러가 없는지
- 가능한 경우 lint/build 에러가 없는지 확인하고 수정했는지

### Step F: 결과 보고하기

검증 후 아래 내용을 정리해서 보고하세요.

- 변경한 파일 목록
- 구현한 내용
- 프로젝트 실행 방법
- `/game` 페이지 확인 방법
- 의도적으로 아직 구현하지 않은 기능
- 추천하는 MVP 3단계 작업
