# MVP 4단계 작업 Command

당신은 1인칭 시점의 3D 멀티플레이 심리 추리 액션 게임을 개발하는 역할입니다.

코드를 수정하기 전에 먼저 아래 문서를 읽으세요.

- docs/game-concept.md
- docs/mvp-scope.md
- docs/technical-direction.md
- docs/mvp-roadmap.md

위 문서들을 프로젝트의 기준 문서로 사용하세요.

## 프로젝트 배경

MVP 3단계에서 player/seeker 키보드 이동, 1인칭 시야 회전(포인터 락), NPC 자동 배회, 그리고 로컬 테스트용 역할 전환 디버그 기능(`R` 키)을 추가했습니다. 이번 MVP 4단계는 그 위에서 술래의 공격과 그 결과(탈락, 체력, 경고 이펙트, 기여도)를 추가합니다.

게임 자체의 콘셉트/역할/규칙은 위에서 읽은 `docs/game-concept.md`, `docs/mvp-scope.md`를 기준으로 삼으세요 (여기서 다시 요약하지 않습니다).

기존 역할 전환 디버그 기능(`src/game3d/debug/*`)은 그대로 재사용하세요. 이번 단계의 공격/피격/체력/경고 UI는 현재 조작 중인 역할(`controlledRole`)에 따라 다르게 보여야 합니다 (예: 체력은 seeker 조작 중일 때, 경고 이펙트/기여도는 player 조작 중일 때).

## 공통 규칙

작업 중 아래 문서의 규칙을 따르세요.

- `CLAUDE.md` — 클라이언트 전용 렌더링, 게임 엔티티/mock 데이터 분리 구조

- 전체 게임을 한 번에 구현하지 않고, 현재 요청받은 MVP 단계의 범위만 구현합니다.
- 구현 후 변경 파일, 실행 방법, 확인 방법을 정리합니다.

## MVP 4단계 목표

이번 작업에서는 MVP 4단계만 구현합니다. (`docs/mvp-roadmap.md` 기준 "술래가 도망자를 찾고 공격한다" 단계입니다.)

MVP 4단계 목표는 다음과 같습니다.

- seeker가 공격 액션을 수행할 수 있습니다 (조작 중인 역할이 seeker일 때만 가능합니다).
- seeker가 hider(player)를 공격하면 해당 hider가 탈락 처리됩니다.
- seeker가 NPC를 공격하면 seeker의 체력이 1 감소합니다.
- seeker의 기본 체력은 5개입니다.
- hider가 seeker 근접 범위 내에 있으면, hider를 조작 중일 때 화면 테두리에 경고 이펙트가 표시됩니다.
- hider가 seeker와 가까울수록 개인 기여도 점수가 오르고, 이 점수는 hider를 조작 중일 때만 표시됩니다.
- 탈락 상태와 seeker 체력 상태가 화면에 표시됩니다.
- 미션, 강제 변신, 최종 승패 플로우(게임 시작/종료), 실시간 서버 판정은 아직 구현하지 않습니다.

## 작업 방식

### Step A: 문서 읽고 요약하기

먼저 아래 문서를 읽으세요.

- docs/game-concept.md
- docs/mvp-scope.md
- docs/technical-direction.md
- docs/mvp-roadmap.md

그다음 아래 항목을 요약하세요.

- 게임의 핵심 콘셉트
- MVP 3단계에서 이미 구현된 내용 (현재 코드 기준)
- MVP 4단계에 포함되는 기능
- MVP 4단계에서 제외되는 기능
- Next.js와 Three.js의 역할 분리
- 구현 시 주의해야 할 점

이 단계에서는 파일을 수정하지 마세요.

### Step B: 작업 계획 제안하기

문서 요약 후 MVP 4단계 구현 계획을 제안하세요.

아래 항목을 포함하세요.

- 작업 범위
- 제외 범위
- 예상 파일 구조(신규/수정 파일)
- 구현 순서
- 리스크 또는 결정이 필요한 부분 (예: 공격 입력 방식과 판정 방식(레이캐스트 vs 거리 기반), 근접 범위/경고 이펙트 표현 방식, 기여도 점수 계산 방식, 탈락한 hider의 처리 방식)

이 단계에서도 파일을 수정하지 마세요.

### Step C: 승인 기다리기

파일을 수정하기 전에 반드시 명시적인 승인을 기다리세요.

아래와 같은 승인 표현이 있을 때만 구현을 시작하세요.

- "진행해"
- "구현해"
- "좋아 진행"
- "Proceed"
- "Implement it"
- "Go ahead"

### Step D: MVP 4단계 구현하기

승인을 받은 뒤 MVP 4단계만 구현하세요.

구현 방향은 다음과 같습니다.

- 기존 MVP 3단계 구조(`src/game3d/*`)를 최대한 재사용하고 확장합니다.
- seeker 공격 입력과 판정 로직을 추가합니다 (조작 중인 역할이 seeker일 때만 동작).
- hider(player) 피격 시 탈락 상태를, NPC 오공격 시 seeker 체력 감소를 상태로 관리합니다.
- seeker 체력, hider 탈락 상태, hider 근접 경고, hider 기여도 점수를 HUD로 표시하되 `controlledRole`에 따라 다른 정보를 보여줍니다.
- 공격, 피격, 체력, 기여도 등 게임 상태는 이후 실시간 서버 상태로 교체할 수 있도록 구조화합니다.
- 미션, 강제 변신, 승패 플로우, 실시간 서버 판정은 이번 단계에서 구현하지 않습니다.
- 이후 단계(미션, 강제 변신, 승패 플로우)를 추가할 수 있는 구조로 만듭니다.

### Step E: 검증하기

구현 후 프로젝트를 검증하세요.

아래 항목을 확인하세요.

- `/game` 페이지가 정상적으로 렌더링되는지
- seeker로 조작 전환 후 공격 액션이 동작하는지
- seeker가 hider를 공격하면 hider가 탈락 처리되는지
- seeker가 NPC를 공격하면 체력이 1 감소하는지
- seeker 체력이 0 이하로 내려가지 않고 화면에 표시되는지
- hider로 조작 전환 후 seeker 근접 시 경고 이펙트가 표시되는지
- hider 기여도 점수가 seeker와의 거리에 따라 변하고, hider 조작 중에만 표시되는지
- 디버그 역할 전환 기능이 여전히 정상 동작하는지
- hot reload 또는 컴포넌트 remount 시 3D 캔버스/렌더러가 중복 생성되지 않는지
- TypeScript 타입 에러가 없는지
- 가능한 경우 lint/build 에러가 없는지 확인하고 수정했는지

### Step F: 결과 보고하기

검증 후 아래 내용을 정리해서 보고하세요.

- 변경한 파일 목록
- 구현한 내용
- 프로젝트 실행 방법
- `/game` 페이지 확인 방법 (공격/피격/체력/경고/기여도 확인 방법 포함)
- 의도적으로 아직 구현하지 않은 기능
- 추천하는 MVP 5단계 작업
