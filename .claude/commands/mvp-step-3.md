# MVP 3단계 작업 Command

당신은 1인칭 시점의 3D 멀티플레이 심리 추리 액션 게임을 개발하는 역할입니다.

코드를 수정하기 전에 먼저 아래 문서를 읽으세요.

- docs/game-concept.md
- docs/mvp-scope.md
- docs/technical-direction.md
- docs/mvp-roadmap.md

위 문서들을 프로젝트의 기준 문서로 사용하세요.

## 프로젝트 배경

MVP 2단계에서 mock 데이터 기반 player/seeker/NPC를 3D 모형으로 표시하고, 1인칭 시점 카메라(자신의 캐릭터는 미표시)를 적용했습니다. 이번 MVP 3단계는 그 위에서 캐릭터 이동과 시야 회전을 추가합니다.

게임 자체의 콘셉트/역할/규칙은 위에서 읽은 `docs/game-concept.md`, `docs/mvp-scope.md`를 기준으로 삼으세요 (여기서 다시 요약하지 않습니다).

## 공통 규칙

작업 중 아래 문서의 규칙을 따르세요.

- `CLAUDE.md` — 클라이언트 전용 렌더링, 게임 엔티티/mock 데이터 분리 구조
- `docs/mvp-scope.md`의 `Excluded` 목록

추가로 아래 두 가지는 이번 커맨드에 고유한 규칙이니 항상 지키세요.

- 전체 게임을 한 번에 구현하지 않고, 현재 요청받은 MVP 단계의 범위만 구현합니다.
- 구현 후 변경 파일, 실행 방법, 확인 방법을 정리합니다.

## MVP 3단계 목표

이번 작업에서는 MVP 3단계만 구현합니다. (`docs/mvp-roadmap.md` 기준 "캐릭터들이 움직인다" 단계입니다.)

MVP 3단계 목표는 다음과 같습니다.

- player와 seeker가 키보드(및 가능하면 터치) 입력으로 이동합니다.
- 마우스(또는 터치 드래그)로 1인칭 시야를 회전할 수 있습니다.
- NPC가 간단한 자동 이동 패턴(예: 랜덤 방향 배회, 경로 왕복 등)으로 움직입니다.
- 모든 캐릭터는 맵 경계 밖으로 나가지 않습니다.
- 역할별 기본 이동 속도를 타입/상수로 정의합니다.
- 1인칭 카메라가 player의 이동과 시야 회전을 따라갑니다(자신의 캐릭터는 계속 화면에 표시하지 않습니다).
- 로컬 테스트 편의를 위해 술래/도망자 역할을 번갈아 조작할 수 있는 디버그 기능을 추가합니다 (아래 "디버그 기능" 참고).
- 공격 판정은 아직 구현하지 않습니다.
- 미션은 아직 구현하지 않습니다.
- 실시간 서버 동기화는 아직 구현하지 않습니다.
- 고급 NPC AI(경로 탐색, 플레이어 추적 등)는 아직 구현하지 않습니다.
- DB 저장은 아직 구현하지 않습니다.

## 디버그 기능 (테스트 전용)

한 명이 술래와 도망자 역할을 번갈아 테스트해야 하므로, 아래 디버그 기능을 함께 추가합니다.

- 특정 키 입력(또는 화면의 디버그 버튼)으로 현재 조작(카메라 + 키보드 이동) 대상을 player ↔ seeker로 전환할 수 있습니다.
- 전환 시 새로 조작하는 엔티티는 화면에서 숨겨지고(자기 캐릭터 미표시 규칙 유지), 이전까지 조작하던 엔티티는 다시 렌더링됩니다.
- 현재 조작 중인 역할을 화면에 표시해 혼동을 방지합니다 (예: 좌상단에 "DEBUG: controlling seeker").
- 이 기능은 `docs/mvp-scope.md`에 정의된 실제 게임 규칙이 아니라 로컬 개발/테스트 전용 도구입니다. 이후 실시간 서버가 도입되면(각자 자신의 역할만 조작) 제거하거나 개발 모드 전용으로 분리할 수 있는 구조로 만듭니다.

## 작업 방식

### Step A: 문서 읽고 요약하기

먼저 아래 문서를 읽으세요.

- docs/game-concept.md
- docs/mvp-scope.md
- docs/technical-direction.md
- docs/mvp-roadmap.md

그다음 아래 항목을 요약하세요.

- 게임의 핵심 콘셉트
- MVP 2단계에서 이미 구현된 내용 (현재 코드 기준)
- MVP 3단계에 포함되는 기능
- MVP 3단계에서 제외되는 기능
- Next.js와 Three.js의 역할 분리
- 구현 시 주의해야 할 점

이 단계에서는 파일을 수정하지 마세요.

### Step B: 작업 계획 제안하기

문서 요약 후 MVP 3단계 구현 계획을 제안하세요.

아래 항목을 포함하세요.

- 작업 범위
- 제외 범위
- 예상 파일 구조(신규/수정 파일)
- 구현 순서
- 리스크 또는 결정이 필요한 부분 (예: 입력 처리 방식, 시야 회전 처리 방식(포인터 락 등), NPC 이동 패턴, 좌표/속도 단위)

이 단계에서도 파일을 수정하지 마세요.

### Step C: 승인 기다리기

파일을 수정하기 전에 반드시 명시적인 승인을 기다리세요.

아래와 같은 승인 표현이 있을 때만 구현을 시작하세요.

- "진행해"
- "구현해"
- "좋아 진행"
- "Proceed"
- "Implement it"
- "Go ahead"

### Step D: MVP 3단계 구현하기

승인을 받은 뒤 MVP 3단계만 구현하세요.

구현 방향은 다음과 같습니다.

- 기존 MVP 2단계 구조(`src/game3d/*`)를 최대한 재사용하고 확장합니다.
- 엔티티 타입에 이동/시야 관련 상태(속도, 방향 등)를 추가합니다.
- 역할별 이동 속도는 상수/설정 값으로 분리합니다.
- player/seeker 이동은 키보드 입력(방향키 또는 WASD)으로 처리합니다.
- 1인칭 시야 회전은 마우스(또는 터치 드래그) 입력으로 처리합니다.
- NPC 이동은 매 프레임 갱신되는 간단한 자동 이동 로직으로 처리합니다.
- 모든 엔티티의 이동은 맵 경계를 벗어나지 않도록 클램프합니다.
- 1인칭 카메라는 player의 위치와 시야를 매 프레임 따라가도록 갱신합니다.
- 디버그용 역할 전환 기능을 추가하고, 실제 게임 로직과 명확히 구분되도록 분리합니다 (예: 별도 디버그 컴포넌트/훅, HUD에 디버그 표시).
- 공격, 미션, 실시간 상태, 고급 AI는 이번 단계에서 구현하지 않습니다.
- 이후 단계(공격, 미션, 실시간 상태)를 추가할 수 있는 구조로 만듭니다.

### Step E: 검증하기

구현 후 프로젝트를 검증하세요.

아래 항목을 확인하세요.

- `/game` 페이지가 정상적으로 렌더링되는지
- 키보드 입력으로 player 또는 seeker가 이동하는지
- 마우스(또는 터치)로 시야를 회전할 수 있는지
- NPC가 자동으로 움직이는지
- 모든 엔티티가 맵 경계 밖으로 나가지 않는지
- 1인칭 카메라가 player의 이동과 시야를 따라가는지
- 디버그 기능으로 player/seeker 조작 대상을 전환할 수 있고, 전환 시 자기 캐릭터 미표시 규칙이 유지되는지
- hot reload 또는 컴포넌트 remount 시 3D 캔버스/렌더러가 중복 생성되지 않는지
- TypeScript 타입 에러가 없는지
- 가능한 경우 lint/build 에러가 없는지 확인하고 수정했는지

### Step F: 결과 보고하기

검증 후 아래 내용을 정리해서 보고하세요.

- 변경한 파일 목록
- 구현한 내용
- 프로젝트 실행 방법
- `/game` 페이지 확인 방법 (이동/시야 회전/디버그 역할 전환 확인 방법 포함)
- 의도적으로 아직 구현하지 않은 기능
- 추천하는 MVP 4단계 작업
