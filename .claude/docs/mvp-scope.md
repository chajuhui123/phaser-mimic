# MVP Scope

첫 번째 MVP의 목표는 재미 검증이다.

핵심 질문:

- NPC 사이에 숨어 있는 플레이어를 찾는 것이 재미있는가?
- 술래가 관찰로 참여자를 찾아낼 수 있는가?
- 참여자가 NPC처럼 위장하는 것이 가능한가?
- 미션 실패 시 강제 변신이 긴장감을 만드는가?
- 한 판을 다시 하고 싶게 만드는가?

## Included

MVP에서는 아래 기능만 구현한다.

- 방 생성/입장
- 최소 인원 기반 게임 시작
- 술래/참여자 역할 배정
- 2D 맵
- 1인칭 시점 카메라
- NPC 다수 생성
- 플레이어 이동
- NPC 자동 이동
- 술래 공격
- NPC 오공격 시 술래 체력 감소
- 참여자 피격 시 탈락
- 간단한 미션 시스템
- 미션 실패 시 N초간 랜덤 생명체로 강제 변신
- 제한 시간
- 승패 판정
- 결과 화면
- 참여 티켓 차감

## Excluded

MVP에서는 아래 기능을 구현하지 않는다.

- 스킨
- 상점
- 랭킹
- 친구 초대
- 복잡한 성장 시스템
- 인앱 결제
- 광고
- 3D 그래픽
- 고급 AI NPC
- 대규모 N:N 팀전
- 음성 채팅
- 실시간 채팅
- 길드/클랜
- 복잡한 매치메이킹

## Initial Game Rules

초기 MVP에서는 아래 규칙을 우선 적용한다.

- 한 판은 N분 동안 진행된다.
- 술래는 1명으로 시작한다.
- 참여자는 1명 이상이다.
- 술래 체력은 5개다.
- 술래가 NPC를 공격하면 체력이 1 감소한다.
- 술래가 참여자를 공격하면 해당 참여자는 탈락한다.
- 참여자는 미션을 실패하면 N초 동안 랜덤 생명체로 강제 변신한다.
- 제한 시간이 끝났을 때 참여자가 1명 이상 살아 있으면 참여자 팀이 승리한다.
- 모든 참여자가 탈락하면 술래 팀이 승리한다.
- 술래 체력이 0이 되면 참여자 팀이 승리한다.

## Ticket System

초기 버전에서는 복잡한 재화 시스템은 구현하지 않는다.

단, 게임 참여에는 참여 티켓이 필요하다.

- 게임 1회 참여 시 티켓 1개를 사용한다.
- 티켓 부족 시 게임에 참여할 수 없다.
- 티켓 지급/충전/구매 방식은 추후 설계한다.
