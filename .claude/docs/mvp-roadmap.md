# MVP Roadmap

이 문서는 게임 MVP를 단계적으로 구현하기 위한 로드맵이다.

각 단계는 이전 단계가 완료된 뒤 진행한다. 한 번에 전체 게임을 구현하지 않고, 현재 단계의 목표만 구현한다.

## MVP Step 1: 화면에 게임이 뜬다

목표:

- `/game` 페이지에서 Phaser 게임 화면이 렌더링된다.
- 간단한 2D 맵 또는 배경이 표시된다.
- 1인칭 시점 카메라가 적용된다 (자신의 캐릭터는 화면에 보이지 않는다).
- mock 데이터 기반으로 player 1명, seeker 1명, NPC 여러 명이 표시된다.
- 각 역할은 시각적으로 구분된다.

제외:

- 캐릭터 이동
- 공격
- 미션
- 실시간 서버
- 데이터베이스

완료 기준:

- `/game` 페이지 진입 시 Phaser 캔버스가 보인다.
- player, seeker, NPC가 화면에 표시된다.
- Phaser 인스턴스가 중복 생성되지 않는다.

## MVP Step 2: 캐릭터들이 움직인다

목표:

- player와 seeker가 키보드 또는 터치 입력으로 이동한다.
- NPC가 간단한 자동 이동 패턴으로 움직인다.
- 맵 경계 밖으로 나가지 않는다.
- 역할별 기본 이동 속도와 상태를 정의한다.

제외:

- 공격 판정
- 미션
- 실시간 서버 동기화
- 고급 NPC AI

완료 기준:

- 사용자가 player 또는 seeker를 움직일 수 있다.
- NPC들이 자동으로 움직인다.
- 캐릭터가 맵 밖으로 벗어나지 않는다.

## MVP Step 3: 술래가 참여자를 찾고 공격한다

목표:

- seeker가 공격 액션을 수행할 수 있다.
- seeker가 hider를 공격하면 hider가 탈락한다.
- seeker가 NPC를 공격하면 seeker의 체력이 1 감소한다.
- seeker의 기본 체력은 5개다.
- 탈락 상태와 체력 상태가 화면에 표시된다.

제외:

- 미션
- 강제 변신
- 최종 승패 플로우
- 실시간 서버 판정

완료 기준:

- seeker가 공격할 수 있다.
- hider 피격 시 탈락 처리된다.
- NPC 오공격 시 seeker 체력이 감소한다.
- seeker 체력이 화면에 표시된다.

## MVP Step 4: 미션과 강제 변신이 동작한다

목표:

- 일정 주기로 hider에게 미션이 주어진다.
- hider가 제한 시간 안에 미션을 수행하면 성공 처리된다.
- hider가 미션을 실패하면 N초 동안 랜덤한 다른 생명체로 강제 변신한다.
- 변신 상태가 화면에서 명확히 보인다.
- 변신 시간이 끝나면 원래 모습으로 돌아온다.

제외:

- 복잡한 미션 종류
- 고급 연출
- 서버 기반 미션 동기화
- 재화 보상

완료 기준:

- 미션이 주기적으로 발생한다.
- 미션 성공/실패가 판정된다.
- 실패 시 강제 변신이 동작한다.
- 변신 종료 후 원래 상태로 복귀한다.

## MVP Step 5: 한 판의 시작/종료/승패 흐름이 완성된다

목표:

- 게임 시작 전 대기 상태를 만든다.
- 시작 카운트다운을 추가한다.
- 제한 시간 타이머를 추가한다.
- 모든 hider가 탈락하면 seeker 팀이 승리한다.
- 제한 시간이 끝났을 때 hider가 1명 이상 살아 있으면 hider 팀이 승리한다.
- seeker 체력이 0이 되면 hider 팀이 승리한다.
- 게임 종료 후 결과 화면을 보여준다.

제외:

- 실제 방/로비 서버
- 랭킹
- 보상 지급
- 티켓 충전/구매
- 장기 전적 저장

완료 기준:

- 한 판이 시작되고 종료된다.
- 승패 조건이 정상적으로 작동한다.
- 결과 화면에서 승리 팀과 주요 결과가 표시된다.
