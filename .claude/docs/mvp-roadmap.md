# MVP Roadmap

이 문서는 게임 MVP를 단계적으로 구현하기 위한 로드맵이다.

각 단계는 이전 단계가 완료된 뒤 진행한다. 한 번에 전체 게임을 구현하지 않고, 현재 단계의 목표만 구현한다.

## MVP Step 1: 3D 렌더링 파이프라인이 동작한다

목표:

- Phaser 관련 패키지/코드를 제거하고 `three`, `@react-three/fiber`, `@react-three/drei`를 설치한다.
- `/game` 페이지에서 빈 3D 씬(바닥 + 기본 조명 + 고정 카메라)이 렌더링된다.
- Three.js 렌더링은 클라이언트 전용으로만 실행되고 SSR 에러가 없다.
- hot reload 또는 컴포넌트 remount 시 3D 캔버스/렌더러가 중복 생성되지 않는다.

제외:

- 게임 엔티티(player/seeker/npc) 표시
- 1인칭 게임 카메라(자기 캐릭터 숨기기 등 게임 로직)
- 이동, 공격, 미션 등 모든 게임플레이 로직

완료 기준:

- `/game` 페이지 진입 시 WebGL 캔버스가 보이고 빈 3D 공간(바닥+조명)이 렌더링된다.
- 콘솔에 SSR/hydration 관련 에러가 없다.
- 페이지를 새로고침하거나 컴포넌트가 remount돼도 캔버스가 중복 생성되지 않는다.
- 기존 Phaser 의존성이 `package.json`에서 제거되어 있다.

## MVP Step 2: 화면에 게임이 뜬다

목표:

- mock 데이터 기반으로 player 1명, seeker 1명, NPC 여러 명이 3D 모형(초기에는 primitive)으로 표시된다.
- 각 역할은 시각적으로 구분된다.
- 1인칭 시점 카메라가 적용된다 (자신의 캐릭터는 화면에 보이지 않는다).

제외:

- 캐릭터 이동
- 공격
- 미션
- 실시간 서버
- 데이터베이스

완료 기준:

- player, seeker, NPC가 화면에 표시된다.
- 각 역할이 형태 또는 색으로 구분된다.
- player 자신의 모델은 화면에 보이지 않는다.

## MVP Step 3: 캐릭터들이 움직인다

목표:

- player와 seeker가 키보드 또는 터치 입력으로 이동한다.
- 마우스(또는 터치 드래그)로 1인칭 시야를 회전할 수 있다.
- NPC가 간단한 자동 이동 패턴으로 움직인다.
- 맵 경계 밖으로 나가지 않는다.
- 역할별 기본 이동 속도와 상태를 정의한다.

제외:

- 공격 판정
- 미션
- 실시간 서버 동기화
- 고급 NPC AI

완료 기준:

- 사용자가 player 또는 seeker를 움직일 수 있다.
- 사용자가 시야를 회전해 주변을 둘러볼 수 있다.
- NPC들이 자동으로 움직인다.
- 캐릭터가 맵 밖으로 벗어나지 않는다.

## MVP Step 4: 술래가 도망자를 찾고 공격한다

목표:

- seeker가 공격 액션을 수행할 수 있다.
- seeker가 hider를 공격하면 hider가 탈락한다.
- seeker가 NPC를 공격하면 seeker의 체력이 1 감소한다.
- seeker의 기본 체력은 5개다.
- hider가 seeker 근접 범위 내에 있으면 hider 화면 테두리에 경고 이펙트가 표시된다.
- hider가 seeker와 가까울수록 개인 기여도 점수가 오르며, 이 점수는 본인에게만 표시된다.
- 탈락 상태와 체력 상태가 화면에 표시된다.

제외:

- 미션
- 강제 변신
- 최종 승패 플로우
- 실시간 서버 판정

완료 기준:

- seeker가 공격할 수 있다.
- hider 피격 시 탈락 처리된다.
- NPC 오공격 시 seeker 체력이 감소한다.
- seeker 체력이 화면에 표시된다.

## MVP Step 5: 미션과 강제 변신이 동작한다

목표:

- 일정 주기로 hider에게 미션이 주어진다.
- hider가 제한 시간 안에 미션을 수행하면 성공 처리된다.
- hider가 미션을 실패하면 5초 동안 랜덤한 다른 생명체로 강제 변신한다.
- NPC도 미션과 같은 행동(점프, 춤 등)을 랜덤 주기로 수행해 hider의 위장을 돕는다.
- 변신 상태가 화면에서 명확히 보인다.
- 변신 시간이 끝나면 원래 모습으로 돌아온다.

제외:

- 복잡한 미션 종류
- 고급 연출
- 서버 기반 미션 동기화
- 재화 보상

완료 기준:

- 미션이 주기적으로 발생한다.
- 미션 성공/실패가 판정된다.
- 실패 시 강제 변신이 동작한다.
- 변신 종료 후 원래 상태로 복귀한다.

## MVP Step 6: 한 판의 시작/종료/승패 흐름이 완성된다

목표:

- 게임 시작 전 대기 상태를 만든다.
- 시작 카운트다운을 추가한다.
- 게임 시작 시 티켓을 5개 소모하고, 티켓이 부족하면 시작할 수 없다.
- 제한 시간(5분) 타이머를 추가한다.
- 상단 HUD에 술래/도망자 프로필과 남은 시간을 표시한다.
- 모든 hider가 탈락하면 seeker 팀이 승리한다.
- 제한 시간이 끝났을 때 hider가 1명 이상 살아 있으면 hider 팀이 승리한다.
- seeker 체력이 0이 되면 hider 팀이 승리한다.
- 게임 종료 후 결과 화면을 보여준다.
- 탈락한 유저는 관전 모드로 전환되며, 관전 대상을 선택할 수 있다.

제외:

- 실제 방/로비 서버
- 랭킹
- 보상 지급
- 티켓 구매(인앱 결제) — 소모/자동 충전 로직은 포함, 결제만 제외
- 장기 전적 저장

완료 기준:

- 한 판이 시작되고 종료된다.
- 승패 조건이 정상적으로 작동한다.
- 결과 화면에서 승리 팀과 주요 결과가 표시된다.
- 티켓이 부족하면 게임을 시작할 수 없다.
